# График готовности зон — дизайн

## Цель

Заполнить свободную область справа от donut-графика полезным представлением готовности ресторана по зонам или должностям.

## Данные

Использовать существующий `model.zones`, который формируется из чек-листов:

- `label` — должность или зона;
- `percent` — доля активных чек-листов;
- `completed` — значение «выполнено из всего»;
- `tone` — `good`, `medium` или `low`.

Backend, API и формат хранения не менять.

## Интерфейс

- Заголовок: «Готовность зон».
- Подзаголовок: «Выполнение чек-листов по направлениям».
- До пяти горизонтальных шкал.
- Каждая строка показывает название, `completed`, процент и цветовую шкалу.
- Цвета: зелёный для `good`, оранжевый для `medium`, красный для `low`.
- Клик по строке или заголовку открывает раздел «Чек-листы».
- При отсутствии зон показать: «Добавьте чек-листы с должностями, чтобы увидеть готовность зон».

## Компоновка

На desktop donut и график зон находятся в одной двухколоночной рабочей поверхности. На узких экранах график зон переносится под donut и список показателей.

## Ограничения

- Сохранить весь существующий функционал donut-графика.
- Не добавлять зависимости.
- Не менять другие страницы кабинета владельца.
