# Resto Control v2 — запись 016

## Страница владельца сервиса

### Назначение
Страница нужна владельцу самого сервиса Resto Control, а не владельцу отдельного ресторана.

Владелец сервиса видит:
- все рестораны на платформе;
- владельцев ресторанов и контакты;
- тарифы;
- сроки доступа;
- пробные периоды;
- статусы оплат;
- счета;
- платёжные поручения;
- закрывающие документы;
- обращения в поддержку;
- реквизиты сервиса для счетов.

### Логика оплаты
Берётся B2B-логика из старого приложения:
- владелец сервиса выставляет счёт ресторану;
- ресторан оплачивает переводом;
- ресторан отмечает «Оплатил»;
- ресторан прикрепляет платёжное поручение, если оно есть;
- владелец сервиса проверяет платёж;
- владелец сервиса подтверждает оплату;
- после подтверждения доступ продлевается;
- появляются закрывающие документы.

### Разделы страницы
- Рестораны;
- Оплаты;
- Реквизиты;
- Техподдержка;
- Создать ресторан.

### Что не добавлять
- финансовую аналитику ресторанов;
- выручку;
- прибыль;
- средний чек;
- food cost;
- онлайн-эквайринг и автосписания на этом этапе.

Это административная страница владельца сервиса для управления клиентами, доступом и оплатами.
