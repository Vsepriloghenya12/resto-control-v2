# Resto Control v2 — запись 019

## Fullstack-связка интерфейса и API

### Подключено к backend API
- сотрудники: список, создание, блокировка / разблокировка, удаление;
- создание сотрудника создаёт User + Membership + Employee;
- запрещена должность «Кладовщик», используется «Клининг»;
- задачи: список, создание, редактирование, удаление, отметка выполнено;
- чек-листы: список, создание, редактирование, удаление, пункты чек-листа;
- оплата ресторана: счета, отметка «Оплатил», прикрепление поручения, сохранение реквизитов, выбор тарифа;
- мобильный план зала: загрузка залов, столов и броней из backend;
- статусы брони: «Пришли по брони», «Гости сели», «Не пришли», «Отменена»;
- тех. заявка сотрудника: создание через центральный плюс в мобильной версии.

### Backend
Добавлен endpoint:
- `GET /api/restaurant`
- `PATCH /api/restaurant`

Расширена логика:
- `POST /api/employees` создаёт пользователя и доступ сотрудника;
- `DELETE /api/employees/:id` удаляет сотрудника, его пользователя, membership и сессии;
- `PATCH /api/bookings/:id/status` обновляет статус брони и связанный статус стола.
