# Live-журнал Resto Control v2 — 003 employees

## Страница
Сотрудники.

## Назначение
Рабочий раздел владельца/управляющего для контроля команды ресторана: должности, текущий статус, задачи, последняя активность и будущая аттестация.

## Зафиксированный функционал
- Страница открывается из левого меню по пункту «Сотрудники».
- Вверху показывается инфографика: всего сотрудников, на смене сейчас, есть проблемы, средняя аттестация.
- Основной формат страницы — таблица, не карточки.
- Таблица содержит: сотрудник, должность, статус, аттестация, сегодня, задачи, последняя активность, действия.
- Столбика «Доступ» нет: доступ в интерфейсе не выбирается отдельно, он определяется должностью.
- Блока «Ожидают приглашения» нет: приглашения в текущей логике не используются.
- Архивации сотрудников нет: в будущем сотрудника можно будет удалить.
- В форме добавления нет комментария и нет отдельного уровня доступа.

## Форма добавления сотрудника
- Имя сотрудника.
- Телефон или email.
- Должность.
- Временный пароль.

## Должности
Официант, старший официант, бармен, старший бармен, повар, хостес, клининг и другие ресторанные должности.

## Аттестация
На странице уже заложено поле аттестации в процентах. В будущем оно будет связано с тестами и обучением сотрудника: меню, ТТК, стандарты сервиса, правила компании и другие блоки.

## Визуал
Shadcn/v0-подход: чистый светлый интерфейс, компактные метрики, таблица с бейджами статусов, progress bar аттестации, боковая форма добавления сотрудника. Без курсива, без финансовых показателей, без лишней декоративности.


## Исправление после локальной проверки

На странице «Сотрудники» убран дублирующий верхний заголовок из общего topbar. Для внутренних разделов панели владельца верхняя строка остаётся только для поиска, уведомлений, профиля и выхода. Название раздела выводится один раз внутри самой страницы.
