# Resto Control v2 — запись 020

## Чистая Railway/PWA-версия и мобильный «План зала»

### Что добавлено
- мобильный экран **«План зала»**;
- переход в план зала из карточки на мобильном «Обзоре»;
- режимы **Столы / Брони**;
- фильтры столов: все, свободные, забронированные, пришли, сели;
- bottom sheet стола/брони с действиями;
- PWA manifest для установки на телефон;
- service worker;
- фирменное лого из старого приложения;
- Railway config для деплоя через GitHub.

### Важные правила
- не использовать отдельные телефонные обёртки вокруг мобильной версии;
- не дублировать меню;
- верхние три точки не используются, меню живёт во вкладке «Ещё»;
- «Брони и гости» в мобильной версии называется **«План зала»**;
- инвентаризация на мобильном «Обзоре» показывается только если она назначена;
- должность «Кладовщик» не используется, вместо неё — **Клининг**.

### Пуш-уведомления
В проект добавлен базовый service worker и обработчик уведомлений для PWA. Для настоящих серверных push-уведомлений нужен backend-модуль подписок и VAPID-ключи или отдельная native-push реализация в Expo/React Native.

### Деплой
Проект подготовлен к Railway:
- `npm run build`
- `npm run start`
- `railway.json`
