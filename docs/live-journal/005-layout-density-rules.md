# Live-журнал Resto Control v2

## Запись 005 — правило плотности и ширины рабочих экранов

Страницы должны нормально помещаться при обычном масштабе браузера 100%.

Если интерфейс приходится уменьшать масштабом браузера до 80%, значит layout сделан неправильно.

Для рабочих таблиц и панелей фиксируется правило:

- не делать лишнюю внешнюю обёртку вокруг большого рабочего блока;
- таблица должна получать максимум доступной ширины;
- правая панель не должна ломать основной контент;
- если ширины не хватает, сначала уплотняются отступы, затем сокращаются подписи, затем срабатывает адаптивный перенос;
- данные не должны обрезаться;
- внешний вид должен оставаться собранным, а не случайно сжатым.

Для страницы «Сотрудники» внешний wrapper таблицы убран: фильтры и таблица стали отдельными рабочими поверхностями, за счёт чего центральный блок получил больше полезной ширины.
