# Live-журнал Resto Control v2

## 004 — Правило по текстам интерфейса

В рабочем интерфейсе нельзя показывать служебные подписи о временном режиме, тестовых данных или техническом состоянии разработки.

Пользователь должен видеть продукт как настоящее рабочее приложение ресторана:
- название заведения без служебных добавок;
- понятные рабочие сообщения;
- без технических пояснений внутри UI;
- без фраз про будущий backend на видимых экранах.

Технические ограничения фиксируются в журнале и задачах разработки, но не выводятся на страницы приложения.
