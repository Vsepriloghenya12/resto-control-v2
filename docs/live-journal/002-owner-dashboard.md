# Resto Control v2 — live-журнал

## Запись 002 — Рабочая панель владельца / управляющего

### Назначение

Вторая страница после авторизации владельца или управляющего — операционная панель ресторана. Страница должна быстро показывать состояние смены: готовность процессов, проблемные зоны, сотрудников на смене, стоп-лист, задачи, брони и инвентаризации.

Это не финансовый dashboard. На странице не должно быть выручки, прибыли, среднего чека, food cost или фейковых денежных потерь.

### Кто видит

- `owner` — владелец ресторана;
- `manager` — управляющий / менеджер.

Сотрудник и старший сотрудник после входа идут в отдельную мобильную рабочую часть. На этом шаге для сотрудника оставлена временная страница-заглушка, чтобы маршрутизация работала.

### Что есть на экране

- левый sidebar с навигацией;
- верхняя панель с приветствием, поиском, уведомлениями, профилем и выходом;
- главный блок «Операционная сводка за сегодня»;
- круговой индикатор готовности смены;
- показатели: чек-листы, задачи, проблемы, сотрудники на смене;
- блок «Готовность по зонам»;
- карточки ключевых операционных показателей;
- блок «Готовность процессов по зонам»;
- блок «Требует внимания»;
- блок «Сотрудники на смене»;
- статус открытой смены внизу sidebar.

### Что убрано после правки

- нижний блок «Быстрые действия», потому что он дублировал левое меню;
- любой наклонный текст / курсив;
- декоративная hero-карточка без инфографики.

### Правило по блокам

Каждый блок на dashboard должен иметь отдельную функцию. Если блок повторяет уже существующее меню или действие, он не нужен.

### Что нельзя добавлять

- revenue;
- sales;
- profit;
- gross margin;
- average check;
- fake money loss;
- финансовые прогнозы;
- декоративные графики без операционного смысла;
- лишние CTA, которые дублируют sidebar.

### Визуал

Ориентир — shadcn/v0 dashboard: чистый sidebar, topbar, аккуратные карточки, badges, компактные списки и инфографика. Стиль адаптирован под Resto Control: рабочее ресторанное приложение, спокойные границы, читаемые данные, без лишнего SaaS-пафоса.

Текст в интерфейсе должен быть обычным, без `italic`.

### Архитектура frontend

Основные файлы:

- `src/pages/owner-dashboard/OwnerDashboardPage.tsx`
- `src/pages/owner-dashboard/OwnerDashboardPage.css`
- `src/pages/employee-start/EmployeeStartPage.tsx`
- `src/pages/employee-start/EmployeeStartPage.css`
- `src/app/App.tsx`

Логика маршрута:

- если сессии нет — `AuthPage`;
- если роль `owner` или `manager` — `OwnerDashboardPage`;
- если роль `senior` или `employee` — `EmployeeStartPage`.

### Проверка сценария

- вход с `owner@resto.local` открывает панель владельца;
- вход с `employee@resto.local` открывает временную страницу сотрудника;
- кнопка выхода возвращает на страницу авторизации;
- первая страница авторизации не сломана;
- `npm run build` проходит.
