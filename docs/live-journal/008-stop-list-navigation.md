# Live-журнал Resto Control v2

## Запись 010 — Стоп-лист не выносится в отдельную вкладку владельца

Решение: отдельная вкладка «Стоп-лист» убрана из левого меню панели владельца.

Причина: стоп-листами должны управлять старшие по подразделениям в своей рабочей части, а владелец/управляющий не должен получать отдельный перегруженный раздел ради этого сценария.

Что остаётся у руководителя:
- на главной панели видно количество позиций в стоп-листе;
- события стоп-листа могут попадать в блок «Требует внимания»;
- отдельная вкладка в меню не нужна.

Правило интерфейса: если процесс выполняется не владельцем, а старшим смены/подразделения, не создавать для владельца отдельную вкладку. Владелец видит сводку и проблемные сигналы на главной странице.
