# Resto Control v2 — запись 017

## Проверка перед мобильной версией

Проверено перед переходом к мобильной части сотрудника.

### Что проверено
- сборка проекта `npm run build` проходит;
- авторизация переводит владельца ресторана в панель владельца;
- авторизация по логинам с `admin`, `super`, `service`, `platform` переводит в страницу владельца сервиса;
- вкладка «Настройки» удалена из меню владельца ресторана;
- вкладки владельца связаны со страницами: Главная, Сотрудники, Чек-листы, Задачи, План зала / Брони, Инвентаризация, ТТК, База знаний, Оплата;
- страница владельца сервиса подключена отдельным маршрутом по роли `service_owner`;
- пробный период при регистрации ресторана задаётся на 14 дней;
- бейдж оплаты на главной больше не жёстко захардкожен датой, а считается от `subscriptionEndsAt`;
- бейдж оплаты показывается только при активной оплате и если до окончания доступа осталось 5 дней или меньше.

### Важное ограничение текущего этапа
Сейчас проект остаётся frontend-прототипом на локальных данных. Навигация и экраны связаны, но сохранение, удаление, загрузка файлов, скачивание документов, подтверждение оплаты и другие бизнес-действия ещё не подключены к backend/API.

Перед production-логикой нужно будет подключить backend-модели и endpoints. Мобильную версию можно начинать как следующий UI/UX-этап, но реальные действия сотрудника должны позже работать через тот же backend.
