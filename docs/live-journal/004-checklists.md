# Live-журнал Resto Control v2

## Запись 004 — Страница «Чек-листы»

Страница «Чек-листы» в панели владельца нужна для создания и настройки шаблонов чек-листов, а не для контроля выполнения. Оперативное состояние чек-листов владелец видит на главной странице.

### Назначение

- создавать чек-листы;
- редактировать чек-листы;
- настраивать пункты;
- задавать должность;
- задавать период выполнения;
- добавлять пример выполнения и фото примера к пунктам;
- включать и выключать чек-листы.

### Типы чек-листов

Только три типа:

- Открытие;
- Закрытие;
- Произвольный.

Бар, зал, кухня, склад и другие направления не являются типами чек-листа. Они задаются через должность или название шаблона.

### Временной период

Каждому чек-листу задаётся период выполнения:

- время начала;
- время окончания.

Сотрудник должен выполнить чек-лист в этот период.

### Пункты чек-листа

У каждого пункта есть:

- текст пункта;
- обязательный / необязательный;
- нужно фото выполнения / не нужно;
- пример выполнения, если нужен;
- фото примера, если нужно.

Фото примера загружает владелец в настройках чек-листа. Фото выполнения делает сотрудник при прохождении чек-листа. Это разные сущности.

### Визуал

Страница сделана как конструктор:

- слева список чек-листов;
- справа редактор выбранного чек-листа;
- сверху общий header страницы без дублирования;
- без инфографики выполнения;
- без лишних таблиц статусов;
- без финансовых показателей;
- без demo-подписей;
- layout должен работать на 100% масштабе браузера.

### Архитектура данных

```ts
type ChecklistType = 'opening' | 'closing' | 'custom'

type Checklist = {
  id: string

  endTime: string

  requiredPhotosCount: number

  updatedAt: string

```

