# Resto Control v2 — шаг 02: авторизация + панель владельца

В этом архиве уже есть две утверждённые части нового приложения:

- первая страница авторизации;
- вторая страница — рабочая панель владельца / управляющего.

## Что внутри

- desktop-авторизация: вход / создание ресторана;
- mobile-вход сотрудника;
- маршрутизация по роли;
- панель владельца с операционной инфографикой;
- временная страница сотрудника, чтобы сценарий входа не ломался;
- структура папок под дальнейшие страницы.

## Как запустить локально

```bash
npm install
npm run dev
```

После запуска открой адрес, который покажет Vite, обычно:

```text
http://localhost:5173
```

## Проверочный вход

Панель владельца:

```text
логин: owner@resto.local
пароль: любой непустой
```

Временная страница сотрудника:

```text
логин: employee@resto.local
пароль: любой непустой
```

## Что важно

`node_modules` и `dist` в архив не добавлены. Их нужно создать локально:

```bash
npm install
npm run dev
```

Backend будет подключён следующим этапом. Сейчас формы и данные используются для проверки страниц и сценариев.

## Зафиксировано по панели владельца

- без курсива;
- с операционной инфографикой;
- без финансовых показателей;
- без нижнего блока «Быстрые действия»;
- без дублирования левого меню;
- стиль — shadcn/v0 по качеству, но под рабочее ресторанное приложение.

## Шаг 03 — Сотрудники

Добавлена страница «Сотрудники» в панели владельца. Пункт «Сотрудники» в левом меню открывает рабочий экран с инфографикой, таблицей сотрудников, аттестацией и формой добавления сотрудника.

Что важно:
- нет блока «Ожидают приглашения»;
- нет столбика «Доступ»;
- нет комментария в форме добавления;
- заложена будущая аттестация сотрудника;
- сотрудники не архивируются, в будущей логике будет удаление.
